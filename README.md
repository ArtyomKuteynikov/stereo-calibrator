# Инструмент стереокалибровки камер

Десктопное GUI-приложение для калибровки самодельных стереокамер по шахматной доске.

Калибровка стереокамеры — задача, которая на первый взгляд кажется сложной: нужно правильно расположить доску, обеспечить хорошее покрытие кадра, отобрать качественные снимки. Это приложение берёт всё это на себя — оно подсказывает, куда переместить доску, предупреждает о проблемах и ведёт вас через весь процесс шаг за шагом. Создано на основе личного опыта и вобрало в себя лучшие практики калибровки.

Написано на Python с использованием OpenCV и PyQt5.

## Возможности

- Автоматическое обнаружение шахматной доски и одновременная съёмка с двух камер
- Советник по позиционированию в реальном времени — подсказывает, куда переместить доску для хорошего покрытия
- Карта покрытия по зонам 3×3 для отслеживания захваченных областей кадра
- Экран проверки снимков — просмотр и отмена неудачных пар перед калибровкой
- Фоновый пайплайн калибровки (стереокалибровка OpenCV + карты ректификации)
- Живой предпросмотр ректифицированной стереопары с эпиполярными линиями после калибровки
- Ручное сохранение: самостоятельно выберите имя файла и директорию
- Отдельная кнопка «Удалить снимки» — очистка папки захвата независимо от сохранения

## Требования

- Python 3.10+
- Две USB-камеры (или любые две камеры, которые OpenCV умеет открывать)
- Распечатанная шахматная доска (см. [Генерация шаблона](#генерация-шаблона))

Установка зависимостей:

```bash
pip install -r requirements.txt
```

## Генерация шаблона

Приложение ожидает стандартную шахматную доску OpenCV (внутренние углы, не квадраты).  
Сгенерировать можно на: https://calib.io/pages/camera-calibration-pattern-generator

**Важно:** в приложении вводится количество **квадратов** (не внутренних углов).  
OpenCV внутренне использует `квадраты − 1` внутренних углов на сторону.

Рекомендуемый начальный шаблон: **8 × 5 квадратов**, размер ячейки **15 мм**.  
Распечатайте на матовой бумаге и наклейте на жёсткую плоскую основу.

## Использование

### 1. Запуск

```bash
python main.py
```

### 2. Экран выбора камер

<!-- SCREENSHOT: экран выбора камер -->

1. Дождитесь окончания определения камер (индикатор загрузки исчезнет).
2. Выберите **левую** и **правую** камеры из выпадающих меню.  
   Под каждым селектором появится живое превью — убедитесь, что выбраны правильные камеры.
3. Задайте параметры калибровки:
   | Параметр | Описание |
   |---|---|
   | **Количество снимков** | Целевое число захватов. Обычно достаточно 25–40. |
   | **Ширина доски (квадраты)** | Количество квадратов по широкой стороне шахматной доски. |
   | **Высота доски (квадраты)** | Количество квадратов по короткой стороне. |
   | **Размер ячейки (мм)** | Физический размер одного квадрата, измеренный линейкой. |
4. Нажмите **«Начать съёмку»** для перехода к фазе захвата.

> Если в папке `images/` остались снимки от предыдущего сеанса, активируется кнопка  
> **«Калибровать по существующим снимкам»** — нажмите её, чтобы перейти сразу к проверке снимков.

---

### 3. Экран захвата

<!-- SCREENSHOT: экран захвата с обнаруженной доской -->

Держите шахматную доску перед обеими камерами.  
Приложение автоматически обнаруживает доску и делает снимок пары, когда обе камеры её чётко видят.

**Индикаторы состояния:**
- `✔ Лев.  •  ✔ Пр.` — доска обнаружена в обеих камерах. Держите неподвижно, захват сработает автоматически.
- `✗ Лев.` / `✗ Пр.` — доска не обнаружена в одной из камер. Скорректируйте угол или освещение.
- Таймер обратного отсчёта показывает время до съёмки.
- После каждого захвата есть короткая пауза для смены позиции.

**Советник по позиционированию (правая панель):**

Приложение анализирует положение доски в кадре и подсказывает, куда переместить её следующей.  
Покрытие отслеживается на **карте зон 3×3**:

| Цвет | Значение |
|---|---|
| Тёмно-серый | Зона ещё не покрыта |
| Синий | 1–2 захвата в зоне |
| Зелёный | 3+ захвата (зона завершена) |

**Советы для хорошей калибровки:**
- Начните с **центра** кадра, сделав 3 снимка под разными углами наклона.
- Затем охватите все 9 зон (углы и края).
- Варьируйте расстояние до камер — часть снимков ближе, часть дальше.
- Держите доску плоско и полностью в поле зрения обеих камер.
- Избегайте размытия движения — замрите до срабатывания захвата.

После достижения целевого числа снимков активируется кнопка **«Запустить калибровку»**.  
Нажмите её для перехода к проверке снимков.

---

### 4. Экран проверки снимков

<!-- SCREENSHOT: экран проверки снимков -->

Все захваченные пары отображаются в виде миниатюр — левый снимок слева, правый справа.

- **Снимите галочку** с любой пары, где есть размытие, частичная видимость доски или несоответствие кадров.
- Используйте **«Выбрать все»** / **«Снять все»** для одновременного переключения всех пар.
- Для продолжения необходимо минимум **4 валидные пары**.
- Нажмите **«Начать калибровку»** по завершении.

---

### 5. Экран результатов

<!-- SCREENSHOT: экран результатов после калибровки -->

Калибровка выполняется в фоне. Прогресс отображается в верхней панели.

По завершении:
- Отображаются **параметры калибровки**: RMS-ошибка, базовое расстояние, матрицы камер, коэффициенты дисторсии, матрицы поворота и трансляции между камерами.
- **Живой ректифицированный предпросмотр** показывает изображения с обеих камер с нарисованными горизонтальными эпиполярными линиями.  
  При хорошей калибровке одинаковые объекты на обоих изображениях должны лежать на одной горизонтальной линии.

**Ориентировочные значения RMS-ошибки:**
| Значение RMS | Качество |
|---|---|
| < 0,5 пкс | Отличное |
| 0,5 – 1,0 пкс | Хорошее |
| 1,0 – 2,0 пкс | Приемлемое |
| > 2,0 пкс | Плохое — требуется перекалибровка |

#### Сохранение результата

<!-- SCREENSHOT: раздел сохранения -->

1. Введите желаемое **имя файла** (без расширения) в текстовое поле.
2. Нажмите **«Обзор…»** для выбора **папки** назначения.
3. Нажмите **«Сохранить результат»** — файл сохранится в формате `.npz`.

Сохранённый файл содержит:

| Ключ | Описание |
|---|---|
| `Q` | Матрица перепроецирования диспаратности в глубину |
| `mapLx`, `mapLy` | Карты ректификации левой камеры |
| `mapRx`, `mapRy` | Карты ректификации правой камеры |

Загрузка в вашем коде:

```python
import numpy as np
import cv2

data = np.load("stereo_calib_data.npz")
frame_left_rect  = cv2.remap(frame_left,  data["mapLx"], data["mapLy"], cv2.INTER_LINEAR)
frame_right_rect = cv2.remap(frame_right, data["mapRx"], data["mapRy"], cv2.INTER_LINEAR)

# Вычисление диспаратности
stereo = cv2.StereoSGBM_create(...)
disparity = stereo.compute(frame_left_rect, frame_right_rect)

# Перепроецирование в 3D
points_3d = cv2.reprojectImageTo3D(disparity, data["Q"])
```

#### Удаление захваченных снимков

Нажмите **«Удалить снимки»** для удаления всех изображений из папок `images/left/` и `images/right/`.  
Это независимо от сохранения — снимки можно удалить до или после сохранения `.npz`.

#### Перекалибровка

Нажмите **«← Перекалибровать»** для возврата к экрану выбора камер и повторной калибровки.

## Решение проблем

**Камеры не обнаружены**  
Убедитесь, что камеры подключены до запуска приложения. На Windows USB-камеры открываются сначала через DirectShow (`CAP_DSHOW`), затем через MSMF как запасной вариант. Попробуйте переподключить камеры и перезапустить приложение.

**Доска не обнаруживается**  
- Убедитесь в равномерном освещении — избегайте бликов и теней на доске.
- При использовании шаблона на мониторе уменьшите яркость экрана.
- Убедитесь, что вся доска видна (не обрезается краями кадра).
- Проверьте, что **настройки размера доски** соответствуют вашему распечатанному шаблону.

**Калибровка завершается ошибкой «нужно минимум 4»**  
Приложению не удалось обнаружить углы шахматной доски в достаточном количестве пар снимков. Повторите захват при лучшем освещении, с более плоской доской и чёткой фокусировкой.

**Высокая RMS-ошибка (> 1,5 пкс)**  
- Используйте больше снимков (40–60), охватывая все зоны.
- Убедитесь, что доска физически плоская (не согнута и не деформирована).
- Увеличьте разнообразие: разные расстояния, углы наклона и позиции.
- Просмотрите снимки и удалите размытые или частично видимые пары.
